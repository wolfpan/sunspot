# Solar Nexus (星冕观测)

**“穿越数据噪音，直击行星级空间天气态势。”**

「星冕观测」是一个轻量级、去中心化的空间天气多源聚合与 AI 态势感知系统。本项目摒弃了传统气象监控面板冗杂的图表堆叠，专注于提取最核心的物理指标，并通过大语言模型（GLM-4）将晦涩的天文数据“转译”为极简、直观的行星环境影响报告。

---

## 理念：从数据堆砌到态势感知 (Philosophy)

有效的数据监控不在于信息的广度，而在于决策的精度。本系统建立在以下三个核心数据处理准则之上：

* **多源互斥与交叉验证 (Cross-Validation):** 舍弃单一数据源的脆弱性，系统化接入 NOAA（美国）、SILSO（比利时）、ESA（欧洲空间局）与 GFZ（德国）的实时接口，确保太阳黑子数、F10.7 射电流量与地磁 Kp 指数的绝对客观与权威。
* **物理影响转译 (Impact Translation):** 拒绝单纯抛出冷冰冰的数字。通过内嵌的 AI 引擎，将 $SSN$、$F10.7$ 和 $Kp$ 的数学矩阵，重构为针对“极光可见度、通信干扰、低轨卫星阻力”的结构化短文本，直接呈现现实物理影响。
* **极简克制 (Radical Minimalism):** 前端 UI 遵循高对比度、低噪音的极简美学。去除所有不必要的交互与视觉冗余，以最高效的排版呈现“源头数据”与“最终结论”。

## 项目概况 (Overview)

本项目采用轻量级的 Node.js + SQLite 全栈架构，专为个人 VPS 或自托管服务器（Self-hosted）的 24/7 静默运行而设计。

* **无感后端流:** 依托 `node-cron` 与异步抓取机制，后端按预定节律（每日 1:00 与 13:00）静默聚合全球基准数据并存入本地 SQLite，彻底斩断对外部大容量数据库的依赖。
* **零延迟响应:** API 接口采用内存缓存穿透设计。AI 分析结果在后台生成后直接驻留内存，前端拉取态势感知报告时实现 0 毫秒级性能损耗。
* **原生开发者支持:** 提供极简的 RESTful API 节点，便于第三方自动化脚本、物联网（IoT）设备或个人终端（如终端看板、智能家居）进行二次调用与集成。

## 核心作用 (Features)

1. **自动化多源同步:** 系统自动对齐并清洗来自不同国家科研机构的异构数据格式，统一提取最精确的时间戳与测量值。
2. **大模型态势计算:** 基于智谱 GLM-4-Flash 模型，根据预设的严苛 Prompt 系统指令，输出不带任何“机器味”与多余修饰的客观环境评估，并自动评定 1-10 级的综合太阳风暴指数。
3. **防假死轮询机制:** 前端具备智能重试与状态检测逻辑。在数据抓取与模型推理的真空期，呈现优雅的加载状态，确保界面永不假死。
4. **沉浸式科普基准:** 前端内置专业的观测指标（SSN, F10.7, Kp）阈值说明与机构背景，建立数据的权威阅读语境。

## 安装方式 (Installation)

系统环境依赖 Node.js 与本地读写权限，部署过程极简，开箱即用。

**1. 克隆项目与安装依赖**

```bash
git clone https://github.com/wolfpan/sunspot.git
cd sunspot
npm install express better-sqlite3 node-cron axios
npm init -y

```

**2. 配置模型密钥 (API Key)**

打开根目录下的 `server.js` 文件，找到配置区，将你的大模型 API 密钥填入对应的常量中（建议在生产环境中将其改造为环境变量读取，以确保凭证安全）：

```javascript
// === 配置区 ===
const GLM_API_KEY = '你的_智谱_API_KEY'; 
// =============

```

**3. 启动服务**

```bash
node server.js

```

系统启动后，将立即执行首次数据同步与数据库（`solar_data.db`）初始化。服务默认运行在 `http://localhost:3000`。建议使用 `PM2` 或 `Docker` 进行进程守护，以保障长效稳定运行。

## 使用方式 (Usage)

1. **数据阅览:** 访问服务器对应端口，即可查看由多源数据构成的聚合矩阵。页面顶部将高亮显示 AI 推演的“综合太阳风暴指数”。
2. **自动化节律:** 开发者无需手动干预。系统内部的 `cron` 调度会接管一切，自动拉取 NOAA 文本、GFZ 的 Kp JSON 以及 ESA 的流量数据。
3. **API 接口调用:**
* **核心数据节点 (GET):** `http://<your-ip>:3000/api/v1/solar/sunspots` (返回包含各机构最新指标记录的 JSON 数组)。
* **态势感知节点 (GET):** `http://<your-ip>:3000/api/v1/solar/analysis` (瞬间返回内存中缓存的最新极简 AI 报告文本)。
